# Кастомные префиксы для владельцев (Owner Custom Prefixes)

## Описание

Эта функциональность позволяет владельцам (owner) userbot'а устанавливать свои собственные префиксы для команд. Каждый владелец может иметь уникальный префикс, который будет работать только для него.

## Установка

1. Убедитесь, что модуль `owner_prefix.py` находится в папке `hikka/modules/`
2. Перезапустите userbot
3. Модуль автоматически загрузится

## Команды

### Основные команды

- `.setownerprefix <префикс>` - Установить кастомный префикс для владельца
- `.removeownerprefix` - Удалить кастомный префикс владельца
- `.myownerprefix` - Показать текущий кастомный префикс владельца
- `.listownerprefixes` - Показать все кастомные префиксы владельцев
- `.ownerprefixsettings` - Открыть настройки функциональности

### Примеры использования

```
.setownerprefix !
!help  # Теперь команда help будет работать с префиксом !

.setownerprefix /
/help  # Теперь команда help будет работать с префиксом /

.removeownerprefix
.help  # Вернулся к стандартному префиксу
```

## Ограничения

1. **Только для владельцев**: Функциональность доступна только пользователям из группы `owner`
2. **Длина префикса**: Максимум 10 символов
3. **ASCII символы**: Только ASCII символы разрешены
4. **Уникальность**: Каждый владелец может иметь только один кастомный префикс

## Настройки

Функциональность можно включить/выключить через команду `.ownerprefixsettings` или через конфигурацию модуля:

```python
self.config["allow_owner_custom_prefix"] = True/False
```

## Технические детали

### Хранение данных

Кастомные префиксы хранятся в базе данных под ключом `owner_prefixes`:

```python
{
    "user_id_1": "!",
    "user_id_2": "/",
    "user_id_3": ">"
}
```

### Обработка команд

Dispatcher проверяет кастомные префиксы в следующем порядке:

1. Получает стандартный префикс из настроек
2. Проверяет, есть ли у отправителя кастомный префикс
3. Если есть и сообщение начинается с кастомного префикса - использует его
4. Иначе использует стандартный префикс

### Поддержка раскладки клавиатуры

Функциональность поддерживает автоматическое переключение раскладки клавиатуры (русская/английская) для кастомных префиксов.

## Безопасность

- Только владельцы могут устанавливать кастомные префиксы
- Префиксы проверяются на валидность (ASCII, длина)
- Нет возможности конфликта с системными командами

## Переводы

Функциональность поддерживает следующие языки:
- Русский (ru)
- Английский (en) 
- Украинский (ua)
- Немецкий (de)

## Устранение неполадок

### Префикс не работает

1. Убедитесь, что вы являетесь владельцем
2. Проверьте, что функциональность включена
3. Убедитесь, что префикс содержит только ASCII символы
4. Проверьте длину префикса (максимум 10 символов)

### Команда не найдена

1. Убедитесь, что используете правильный префикс
2. Проверьте, что команда существует
3. Убедитесь, что у вас есть права на выполнение команды

### Модификации dispatcher

В `hikka/dispatcher.py` добавлена логика проверки кастомных префиксов в методе `_handle_command`:

```python
# Check for owner custom prefix
prefix = default_prefix
if hasattr(event, "sender_id") and event.sender_id:
    owner_prefixes = self._db.get(main.__name__, "owner_prefixes", {})
    if str(event.sender_id) in owner_prefixes:
        owner_prefix = owner_prefixes[str(event.sender_id)]
        if owner_prefix and event.message.message.startswith(owner_prefix):
            prefix = owner_prefix
```
